# HPGLペン設定無視とDXF ByLayer化 設計

## 背景

現行実装はHPGLの`SP`命令および`PE`命令内のペン変更をDXF entityのACI色へ変換している。ペン変更時には連続線を分割し、ペン番号を共通図形データの`color`へ保存し、DXFライターが各entityへgroup code 62として直接出力する。

今後はHPGL側のペン設定を使用せず、CAD側でレイヤーごとに色、線種、線幅を管理する。すべてのDXF entityを`ByLayer`とし、HPGLのペン変更が図形構造、診断、DXF出力へ影響しないようにする。

## 目的と完了条件

- 入力ファイルごとのDXFレイヤー構成を維持する。
- 全entityのColorを`ByLayer`にする。
- HPGLの`SP`および`PE`内のペン変更を完全に無視する。
- ペン変更によって連続線を分割しない。
- ペン番号の値や書式を理由とするエラーを出さない。
- 描画座標、頂点順、レイヤー、AutoCAD 2000互換構造を変更しない。
- ペン変更による分割がなくなる結果として、entity数および`LINE`と`LWPOLYLINE`の構成が変わることは許容する。

## 検討した方法

### 共通図形データからペン色を削除する

HPGLパーサーからペン色状態を削除し、共通図形データにも`color`を持たせない。DXFライターはentityのgroup code 62を出力せず、DXF既定動作によって`ByLayer`にする。不要なペン由来状態が残らず、今回の要件をデータモデルから保証できるため採用する。

### DXF出力時だけペン色を無視する

パーサーと共通図形データにはペン色を残し、DXFライターだけがgroup code 62を省略する。この方法ではペン変更による線分割や不正ペン番号の診断が残り、「ペン設定を完全に無視する」という要件を満たさないため採用しない。

### entityへgroup code 62の値256を出力する

値256で`ByLayer`を明示できるが、entityでgroup code 62を省略した場合も既定値は`ByLayer`である。不要なtagと専用検証が増えるため採用しない。

## HPGLパーサー

### `SP`命令

`SP`を認識済みのno-op命令として扱う。引数の解析、個数検証、整数・範囲検証、連続線の確定、状態変更、診断追加を行わない。

そのため、`SP1`、`SP0`、`SP256`、`SPX`、引数なしの`SP`はいずれも後続の作図へ影響せず、ペン設定を理由とするエラーにもならない。命令の終端や後続命令のtokenizeは従来どおり行う。

### `PE`命令内のペン変更

`PE`デコーダーが返す`pen`イベントは読み飛ばす。連続線の確定、色状態の変更、値の範囲検証、診断追加を行わない。ペンイベントの前後にある座標は、ペンアップ状態や座標モードが同じなら1本の連続線として蓄積する。

`PE`全体のバイト列を解読できない場合は、座標データも信頼できないため従来どおり`PE`命令のエラーとして扱う。これはペン番号の意味検証ではなく、圧縮データ構造の解読失敗である。

### 共通図形データ

パーサー状態から`color`を削除し、線、連続線、円、円弧、文字の共通メタデータからも`color`を削除する。図形は引き続き次の情報を持つ。

- 図形種別と図形固有データ
- DXFレイヤー名
- 入力ファイル名
- 元命令位置

## DXFライター

全entityの共通tagからgroup code 62を削除する。group code 8のファイル別レイヤーは維持する。entity固有の色を指定しないことで、Colorはレイヤー設定を継承する`ByLayer`になる。

LAYER tableでは、レイヤー自体の既定色ACI 7と線種`CONTINUOUS`を維持する。利用者はBricsCADで各レイヤーのプロパティを変更し、所属entityへ一括反映できる。

現行ライターはentity固有の線種group code 6および線幅group code 370を出力していない。この状態を維持するため、線種と線幅も引き続き`ByLayer`になる。

`validateColor`とgeometryの`color`検証は削除する。それ以外のレイヤー、座標、半径、角度、文字、handle、owner、subclass markerの検証は維持する。

## エラー処理

- `SP`の引数は内容にかかわらず診断対象にしない。
- `PE`内の解読可能なペン変更値は内容にかかわらず診断対象にしない。
- `PE`データ全体が構造的に解読不能な場合は従来どおりエラーにする。
- ペン以外のHPGL命令およびDXF geometry検証は変更しない。

## テスト

### HPGLパーサー

- 異なる`SP`値の前後で図形に色属性が付かない。
- `SP`で連続線が分割されない。
- `SP0`、範囲外、非数値、引数なしでもペン由来診断が発生しない。
- `PE`内の複数のペン変更で連続線が分割されない。
- `PE`内のペン値が0または従来の範囲外でもペン由来診断が発生しない。
- 構造的に壊れた`PE`は従来どおり診断され、後続命令から回復する。

### DXFライター

- `LINE`、`LWPOLYLINE`、`CIRCLE`、`ARC`、`TEXT`の各entityにgroup code 62が存在しない。
- group code 8のファイル別レイヤーが維持される。
- LAYER tableの既定色ACI 7が維持される。
- geometry入力に`color`がなくても生成できる。
- 既存のAutoCAD 2000構造、handle、owner、subclass marker検証が通る。

### 結合・実機確認

- 異なる`SP`を含む複数HPGLを変換しても、全entityが`ByLayer`になる。
- ペン変更を除く描画座標、頂点順、レイヤーが期待値と一致する。
- entity数および`LINE`と`LWPOLYLINE`の構成差が、ペン変更境界の統合だけで説明できる。
- BricsCAD V24のプロパティでentityのColorが`ByLayer`と表示される。
- BricsCAD V24でレイヤー色を変更すると、所属entityの表示色が追従する。
- 全VitestとVite本番ビルドを実行する。

## 変更しない範囲

- 入力ファイルごとのレイヤー命名規則
- 座標変換と40 HPGL単位＝1 mmの換算
- 円、円弧、文字、ポリラインの変換規則
- AutoCAD 2000互換DXFのhandle、owner、subclass構造
- UI、Worker、出力ファイル名、ダウンロード動作
- HPGLまたはDXFの外部送信を行わない方針
