# HPGLビューワー・差分表示 設計

## 1. 目的

追加された複数のHPGLファイルをブラウザ内で自動解析し、入力ファイルごとに色分けして重ね合わせ表示する。任意の2ファイルについて、共通図形と各ファイル固有の図形を区別する差分表示も提供する。

HPGLデータと解析結果は外部へ送信しない。既存のDXF変換、診断、ダウンロード契約は維持し、プレビュー処理の失敗がDXF変換を妨げない構成とする。

## 2. 対象範囲

### 2.1 通常表示

- 対応HPGLファイルが追加された直後にプレビュー解析を開始する。
- 全ファイルの図形を同一座標系へ重ね合わせ、ファイルごとに異なる色で描画する。
- 各ファイルの凡例、割り当て色、表示ON/OFFを提供する。
- マウスホイールによるカーソル位置中心の拡大縮小、ポインタードラッグによる移動、全体表示への復帰を提供する。
- HPGLと同じY軸上向きの座標系を、CanvasのY軸下向き座標系へ描画時だけ変換する。

### 2.2 差分表示

- 比較元Aと比較先Bとして、追加済みファイルから異なる2ファイルを選択する。
- 共通図形を薄いグレー、Aだけに存在する図形を青、Bだけに存在する図形をオレンジで描画する。
- Aのみ、共通、Bのみの図形数を表示する。
- 同じ図形が複数存在する場合は個数を保った多重集合として比較する。

### 2.3 対象外

- DXFファイルの読み込みまたはDXF内容の再描画
- HPGLの線幅、線種、CTB、塗りつぶし、ハッチング、ラスターデータの再現
- CAD固有フォントと文字配置の完全再現
- 図形の編集、スナップ、寸法計測、印刷、PDF出力
- 3ファイル以上を同時に用いた差分集合の計算

## 3. 採用方式

プレビュー解析は専用Web Workerで実行し、表示はCanvas 2Dを使用する。既存の`parseHpgl`と共通図形データを再利用し、描画ライブラリや通信を行う外部依存は追加しない。

SVGは図形単位のDOM要素数が大きくなりやすく、実運用の複数図面を重ねた場合に操作性能が低下する可能性があるため採用しない。メインスレッドでのHPGL解析も、大きな入力でUIを停止させるため採用しない。

DXF変換用Workerとプレビュー用Workerは分離する。プレビューはファイル追加・削除時に自動実行される一方、DXF変換は既存の変換ボタンから明示的に実行する。この分離により、プレビューをキャンセルまたは失敗してもDXF変換を実行できる。

## 4. モジュール構成

### 4.1 `src/viewer/geometry.js`

共通図形データを表示と比較へ利用するための純粋関数を提供する。

- 複数図形の描画範囲を算出する。
- 図形を`0.001 mm`精度で正規化し、比較キーを生成する。
- 2つの図形配列を多重集合として比較し、`onlyA`、`common`、`onlyB`へ分類する。
- レイヤー名、ファイル名、元命令位置は比較キーに含めない。

### 4.2 `src/viewer/canvas-renderer.js`

Canvas、表示対象の図形群、表示色、ビューポートを受け取り、描画だけを行う。

- 線と連続線はCanvasのパスで描画する。
- 円と円弧はCanvasの円弧APIで描画する。
- 文字は挿入位置、高さ、回転を反映した簡易テキストとして描画する。
- Canvasの表示サイズと`devicePixelRatio`を考慮し、高DPI画面でも線がぼやけないようにする。
- ビューポートは中心座標と拡大率を保持し、全体表示、拡大縮小、移動を計算する。

### 4.3 `src/viewer/preview-client.js`

プレビュー用Workerを起動し、ファイルとレイヤー名を送信する。戻り値は`promise`と`cancel()`を持つジョブとし、既存の変換クライアントと同じ利用形にそろえる。

完了結果は次の形とする。

```js
{
  files: [{
    name,
    layerName,
    geometries,
    geometryCount,
    errorCount,
    warningCount,
    diagnostics,
  }],
}
```

### 4.4 `src/viewer/preview.worker.js`

ファイルを追加順に読み込み、ファイルごとに`parseHpgl`を呼び出す。1ファイルの読み込みまたは解析が失敗しても、ほかのファイルの解析を続行する。DXF文字列は生成しない。

### 4.5 `src/app.js`

既存のファイル管理と変換状態に、プレビューの状態と画面操作を追加する。

- ファイル追加・削除後にプレビュー解析を自動開始する。
- 実行中の古いプレビュージョブをキャンセルする。
- ジョブごとのトークンを照合し、キャンセル後に遅れて届いた結果を無視する。
- 通常表示のファイル表示状態、差分モード、比較対象A/B、ビューポートを管理する。
- Canvasの描画要求を`requestAnimationFrame`単位にまとめる。
- アプリ破棄時にWorker、イベントリスナー、未実行の描画要求を解放する。

## 5. 画面設計

ファイル一覧と出力設定の間へ「プレビュー」パネルを配置する。

### 5.1 パネルヘッダー

- 見出しは「プレビュー」とする。
- 右側へ「通常表示」「差分表示」の切り替えと「全体表示」ボタンを置く。
- ファイルがない場合はパネルを空状態として表示する。
- 自動解析中は`aria-live`の状態文と進捗を表示する。

### 5.2 通常表示コントロール

- ファイル追加順に、色見本、表示チェックボックス、ファイル名、図形数を表示する。
- 色は白背景と濃色背景の双方で区別できる固定パレットから順番に割り当てる。
- すべてのチェックを外した場合はCanvasを空表示にするが、解析結果は保持する。

### 5.3 差分表示コントロール

- 比較元Aと比較先Bのセレクトボックスを表示する。
- 2ファイル以上ある場合だけ差分表示へ切り替えられる。
- 初期選択は追加順の先頭2ファイルとする。
- AとBに同じファイルを選ぼうとした場合は、もう一方を次に選択可能なファイルへ自動変更する。
- 差分の色と件数を凡例として表示する。

### 5.4 Canvas操作

- 初回解析完了時と表示対象の変更で、表示対象全体が収まるよう自動調整する。
- ホイール操作はカーソル下の図面座標を固定したまま拡大縮小する。
- 主ボタンのドラッグで表示を移動する。
- 「全体表示」で現在の表示対象へ再フィットする。
- 図形がない場合は「表示できる図形がありません」とCanvas領域内へ表示する。

## 6. 図形の同一性

数値は比較前に`0.001 mm`単位へ丸め、`-0`は`0`として扱う。

- `line`: 2端点を辞書順に並べ、始点と終点が逆でも同一とする。
- `polyline`: 正順の頂点列と逆順の頂点列を比較し、辞書順で小さい方をキーにする。
- `circle`: 中心と半径を比較する。
- `arc`: 中心、半径、正規化した開始角度、終了角度を比較する。向きが異なる円弧は同一としない。
- `text`: 挿入点、文字列、高さ、正規化した回転角度を比較する。

比較キーごとにAとBの図形をキューとして保持し、両方に存在する最小個数を`common`へ分類する。余った個数をそれぞれ`onlyA`、`onlyB`へ分類する。分類結果では元の図形データと入力順を保持する。

## 7. 描画範囲と表示

- 線と連続線は全頂点を範囲へ含める。
- 円は中心から半径分の矩形を範囲へ含める。
- 円弧は開始点、終了点、および掃引範囲に含まれる0、90、180、270度の点を範囲へ含める。
- 文字は挿入点を最低限の範囲として扱い、文字高に基づく余白を加える。
- 全体表示にはCanvas端から5%相当、最低12 CSS pxの余白を設ける。
- 単一点または幅・高さが0の図形でも有限の拡大率を使用する。

通常表示ではファイル色を使用し、重なりを認識できる不透明度で描画する。差分表示では通常表示のファイル色を使わず、共通、Aのみ、Bのみの固定色を使用する。

## 8. 状態遷移とエラー処理

- ファイル0件ではプレビュー結果を破棄し、実行中ジョブをキャンセルする。
- ファイル追加・削除では現在の結果を古い状態として非表示にし、新しい解析を開始する。
- 新しい解析開始時に前のジョブをキャンセルする。
- キャンセルは通常の状態遷移として扱い、エラーバナーを表示しない。
- 個別ファイルの読み込み・解析失敗はそのファイルのエラーとして表示し、正常ファイルは描画する。
- Worker自体の失敗はプレビューパネルだけへ表示し、ファイル一覧とDXF変換操作は維持する。
- プレビュー結果の図形配列はDXF変換結果とは共有せず、変換時は既存処理が入力ファイルを改めて解析する。

## 9. アクセシビリティ

- モード切り替え、全体表示、ファイル表示、比較対象選択はキーボードで操作可能なネイティブ要素を使用する。
- Canvasには内容を説明するラベルを付け、差分件数と解析状態はCanvas以外のテキストでも提供する。
- 色だけに依存せず、ファイル名と「Aのみ」「共通」「Bのみ」のラベルを併記する。
- `prefers-reduced-motion`環境でも必要な操作を妨げない。

## 10. テスト方針

### 10.1 図形処理

- 各図形種別の描画範囲と複合範囲
- 空配列、単一点、0幅・0高さでも有限となる全体表示
- `0.001 mm`丸めと`-0`の正規化
- 逆向きの線と連続線の一致
- 円弧方向の相違
- 文字内容、位置、高さ、回転の相違
- 重複図形を含む多重集合差分

### 10.2 Canvasレンダラー

- CSSサイズと`devicePixelRatio`に応じたバッファサイズ
- HPGL座標からCanvas座標へのY反転
- カーソル位置を保つ拡大縮小
- ドラッグ移動と全体表示
- 通常色と差分色の描画選択

Canvas APIは描画命令の境界だけを最小限に記録し、図形処理と状態計算は実データを使う純粋関数として検証する。

### 10.3 WorkerとUI

- ファイルごとの解析結果、診断、図形配列の返却
- 一部ファイルの読み込み失敗後も後続を処理すること
- 完了、Worker失敗、キャンセル、古い結果の無視
- ファイル追加直後の自動解析
- ファイル削除後の再解析
- 通常表示の表示ON/OFF
- 2ファイル未満では差分表示が無効になること
- A/B選択、差分件数、空状態、解析エラー表示
- アプリ破棄時のリソース解放

### 10.4 全体検証

- `npm.cmd test`
- `npm.cmd run build`
- `git diff --check`
- `reference`内の代表HPGLを使用した解析結果と有限な表示範囲の確認
- 最新のEdgeまたはChromeで、複数ファイルの色分け、拡大縮小、移動、全体表示、2ファイル差分を目視確認

## 11. 完了条件

- 対応ファイル追加後、明示操作なしでプレビュー解析が始まる。
- 複数ファイルを色分けして重ね合わせ、ファイル単位で表示を切り替えられる。
- 任意の異なる2ファイルについて、共通、Aのみ、Bのみを`0.001 mm`精度の多重集合比較で分類できる。
- 大きな入力の解析をWorkerで実行し、Canvasで拡大縮小と移動ができる。
- プレビュー失敗時も既存のDXF変換を実行できる。
- 実行時依存、CDN、外部通信を追加しない。
- 自動テスト、ビルド、代表HPGLによる検証を完了する。
