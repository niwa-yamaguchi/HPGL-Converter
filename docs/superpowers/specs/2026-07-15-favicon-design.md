# ファビコン設計

## 目的

HPGL → DXF Converterをブラウザのタブやブックマークから識別しやすくするため、既存画面の配色と変換機能を反映したファビコンを追加する。

## ビジュアル

- 選定案は「B. 変換矢印」とする。
- 背景は既存画面の主要アクセントに合わせた青（`#146fae`）の角丸正方形とする。
- 中央に白い2本の水平矢印を上下逆向きに配置し、入力から出力への変換を表す。
- 上側矢印の始点に黄（`#f1a72d`）、下側矢印の終点に濃紺（`#0c223d`）の丸を置く。
- 16px表示でも輪郭が潰れないよう、文字や細かい装飾は使わず、線端と角は丸くする。

## ファイル構成

- ファビコン本体は `public/favicon.svg` に置く。
- SVGは `viewBox="0 0 64 64"` を持ち、サイズ指定なしで拡縮可能にする。
- `index.html` の `<head>` に `<link rel="icon" type="image/svg+xml" href="/favicon.svg">` を追加する。
- Viteの `public` ディレクトリ配信を利用し、アプリケーションのJavaScriptには依存させない。

## 動作

ページ読み込み時にブラウザが `/favicon.svg` を直接取得する。ファビコンの読み込みに失敗しても変換機能には影響せず、追加のエラー表示やフォールバック処理は設けない。

## 検証

- `index.html` にSVGファビコンへの参照があることを自動テストで確認する。
- SVGに64×64の座標系、青い角丸背景、白い双方向矢印、黄と濃紺の端点が含まれることを確認する。
- `npm run build` 後に `dist/favicon.svg` が生成されることを確認する。
- ブラウザのタブで16px相当の表示を目視確認し、矢印と背景が識別できることを確認する。

## 対象外

- `.ico` やPNG形式の追加
- OS別・テーマ別のファビコン
- Apple Touch IconやWeb App Manifest
- 既存画面のロゴ、見出し、配色の変更

## 完了条件

- サイトを開いたブラウザタブに選定したファビコンが表示される。
- ファビコン追加後も既存テストとViteビルドが成功する。
- ファビコン関連の変更が変換処理の挙動に影響しない。
