# Favicon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** ブラウザのタブでHPGL → DXF Converterを識別できる、承認済みの「変換矢印」SVGファビコンを追加する。

**Architecture:** Viteの `public` ディレクトリに依存関係のないSVGを置き、`index.html` の `<head>` からルート相対URLで参照する。静的ファイルとHTML参照をNode環境のVitestで検証し、Viteビルドで成果物へのコピーも確認する。

**Tech Stack:** HTML5、SVG、Vite 8、Vitest 4、Node.js 22.12以上

## Global Constraints

- 選定デザインは「B. 変換矢印」とする。
- 背景色は `#146fae`、矢印は白、端点は `#f1a72d` と `#0c223d` を使う。
- SVGは `viewBox="0 0 64 64"` とし、16px表示で識別できる単純な図形にする。
- `.ico`、PNG、Apple Touch Icon、Web App Manifestは追加しない。
- アプリケーションのJavaScriptと既存画面の表示・変換処理は変更しない。
- ユーザーの未コミットファイルには触れない。

## File Structure

- Create: `public/favicon.svg` — 青い角丸背景、白い双方向矢印、黄と濃紺の端点を持つファビコン本体。
- Modify: `index.html` — ブラウザにSVGファビコンを通知する `<link rel="icon">` を追加。
- Create: `tests/ui/favicon.test.js` — HTML参照とSVGの必須構造・色を検証。

### Task 1: SVGファビコンの追加とHTML参照

**Files:**
- Create: `public/favicon.svg`
- Modify: `index.html:7-8`
- Test: `tests/ui/favicon.test.js`

**Interfaces:**
- Consumes: ViteがルートURL `/favicon.svg` へ公開する `public` ディレクトリ規約。
- Produces: `GET /favicon.svg` で取得でき、`<link rel="icon" type="image/svg+xml" href="/favicon.svg">` から参照されるSVGファビコン。

- [ ] **Step 1: HTML参照とSVG構造を検証する失敗テストを書く**

`tests/ui/favicon.test.js` を次の内容で作成する。

```js
import { readFile } from 'node:fs/promises';
import { resolve } from 'node:path';
import { describe, expect, it } from 'vitest';

const readProjectFile = path => readFile(resolve(path), 'utf8');

describe('favicon', () => {
  it('links the SVG favicon from the document head', async () => {
    const html = await readProjectFile('index.html');

    expect(html).toContain(
      '<link rel="icon" type="image/svg+xml" href="/favicon.svg">',
    );
  });

  it('uses the approved conversion-arrow SVG design', async () => {
    const svg = await readProjectFile('public/favicon.svg');

    expect(svg).toContain('viewBox="0 0 64 64"');
    expect(svg).toContain('fill="#146fae"');
    expect(svg).toContain('stroke="#ffffff"');
    expect(svg).toContain('fill="#f1a72d"');
    expect(svg).toContain('fill="#0c223d"');
    expect(svg.match(/<path\b/g)).toHaveLength(2);
    expect(svg.match(/<circle\b/g)).toHaveLength(2);
  });
});
```

- [ ] **Step 2: 対象テストを実行して期待どおり失敗することを確認する**

Run: `npm test -- tests/ui/favicon.test.js`

Expected: 2テストがFAILする。HTMLテストはファビコン用 `<link>` がないため失敗し、SVGテストは `public/favicon.svg` が存在しないため `ENOENT` で失敗する。

- [ ] **Step 3: 最小限のSVGとHTML参照を実装する**

`public/favicon.svg` を次の内容で作成する。

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 64 64">
  <title>HPGL to DXF conversion arrows</title>
  <rect width="64" height="64" rx="14" fill="#146fae"/>
  <path d="M14 23h29l-7-7" fill="none" stroke="#ffffff" stroke-width="6" stroke-linecap="round" stroke-linejoin="round"/>
  <path d="M50 41H21l7 7" fill="none" stroke="#ffffff" stroke-width="6" stroke-linecap="round" stroke-linejoin="round"/>
  <circle cx="14" cy="23" r="4" fill="#f1a72d"/>
  <circle cx="50" cy="41" r="4" fill="#0c223d"/>
</svg>
```

`index.html` の `<title>` の直前に次の1行を追加する。

```html

```
